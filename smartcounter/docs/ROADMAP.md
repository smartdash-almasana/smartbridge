# Roadmap

## Fase 0 - Apertura documental

- definir alcance de SmartCounter
- fijar limites explicitos (que es y que no es)
- alinear vocabulario operativo para desarrollo

## Fase 1 - Diccionario y taxonomia

- crear diccionario de campos canonicos contables
- definir taxonomia inicial de tipos de planilla
- establecer reglas base de normalizacion por campo

## Fase 2 - Clasificacion y mapping

- implementar clasificacion de tipo de planilla
- implementar mapping columna original -> campo canonico
- registrar confianza, conflictos y razones

## Fase 3 - Revision asistida y memoria por cliente

- crear flujo de revision humana asistida
- guardar decisiones de mapping por cliente
- reutilizar memoria operativa en archivos futuros del mismo cliente

## Fase 4 - Modulos contables posteriores

- explorar modulos posteriores de valor (por ejemplo, LSD)
- mantener separacion por vertical
- sin promesa de implementacion actual en esta fase documental
