# .airules.md

## Global production policy
This repository is operated as a deterministic SaaS factory.
All agents must preserve continuity, minimize drift, and work from confirmed repo state.

## Execution policy
- Work from the current confirmed repo state only.
- Do not redesign the architecture unless explicitly requested.
- Do not open a new front before the current one is functionally closed.
- Prefer minimal, verifiable, merge-ready changes.
- Prefer one concrete step at a time.

## Agent specialization
### Gemini
Primary responsibilities:
- absorb long context
- audit repo state
- compare docs, code, and prior validated decisions
- detect continuity breaks
- identify the exact point of reentry
- generate compressed professional prompts for other agents

Gemini should be used before Codex on any task that requires broad reading or repo understanding.

### Codex
Primary responsibilities:
- make precise code edits
- create narrow tests
- produce minimal diffs
- validate localized behavior

Codex must not be used as the default repo auditor or broad planning agent.

### Claude
Primary responsibilities:
- review architecture consistency
- review diffs
- detect real risks
- review prompts, contracts, and skills

Claude is a reviewer, not the default implementation engine.

## Context compression policy
Before handing work to another agent:
- keep only confirmed facts
- include only relevant files
- define one goal
- define hard constraints
- define acceptance criteria
- force a structured output format

Remove:
- repeated project history
- general theory
- broad strategy not needed for the current step
- unrelated modules

## Repo continuity policy
Before proposing implementation:
1. verify the active front is actually present in current runtime
2. detect missing deltas between validated work and current branch
3. restore continuity first if broken
4. only then move to the next phase

If continuity is broken, implementation of the next layer is blocked.

## SmartPyme project invariants
- SmartCounter / SmartBridge / SmartSeller are an operating system for SMEs, not dashboards.
- Every finding must contain:
  - specific entity
  - quantified difference
  - explicit comparison of sources
- The system must block under uncertainty.
- The system must not hallucinate.
- The system learns only from human-validated decisions.
- Separation is mandatory between:
  - universal core
  - configurable policy per SME
  - guidance / copilot layer

## Implementation constraints
- Do not invent repo state, business facts, recipients, or results.
- If something is not confirmed, write `no confirmado`.
- Keep existing contracts unless the task explicitly requires a contract change.
- Preserve tenant isolation.
- Avoid side effects unless explicitly required.
- Reuse existing services when possible.
- Do not add infrastructure unless strictly necessary.
- No Celery, Redis, queues, or schedulers unless explicitly requested.

## Prompt quality policy
All generated prompts must be:
- professional
- specific
- executable
- constrained
- scoped to one task
- grounded in confirmed context

Every implementation prompt must include:
- technical role
- single objective
- confirmed context
- allowed files
- hard constraints
- acceptance criteria
- required response format

## Review policy
Every reviewed change must be checked for:
- continuity with the active front
- contract preservation
- side effects
- architectural drift
- real risks only

## Current strategic direction
At the current stage:
- do not keep expanding ingestion plumbing unless there is a real blocker
- prioritize turning business criteria into executable logic
- the next high-value layer is:
  - catalog / rule layer
  - deterministic evaluator
  - actionable findings output

This move is allowed only after the active front is functionally restored.

## Recurring correction policy
- If the same mistake appears twice, update the persistent rules.
- If an agent starts re-auditing closed work without evidence of regression, stop and compress context.
- If Codex is being used for broad discovery, reroute that work to Gemini.
- If a validated front is missing from `main`, restore it before any new phase begins.
